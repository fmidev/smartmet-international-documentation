# Introduction to App Store and Play Store

The development, deployment and delivery of the application requires certain steps before it is possible to have the application in stores for the users. The initial stages require target country to do things before it is possible to start the development.

Both stores update their licence and application version requirements once in a while so account holder must be following these updates, especially legal requirements. This guide and all other FMI provided guides related to either store can be obsolete due to UI and other changes done by Apple or Google.

## App Store

App Store is Apple's application marketplace for IOS devices. The following steps needs to be done by target country:

1) Enroll your organization to Apple Developer Program ([Become a member - Apple Developer Program](https://developer.apple.com/programs/enroll/))
2) Enrolling includes creating an account holder -account. This account is the only account that can do some things so it is an important account. Admin rights are not enough for certain actions.
3) Once organization is created target country should invite FMI developer(s) to the organization as admins.
4) FMI developers with admin rights can then continue with creating the app and adding required certificates etc.

## Play Store

Play Store is Google's application marketplace for Android devices. The following steps needs to be done by target country:

1) Create a Play Console developer profile for you organization ([Required information to create a Play Console developer account - Play Console Help](https://support.google.com/googleplay/android-developer/answer/13628312?hl=en))

2) Enrolling includes creating an account holder -account. This account is the only account that can do some things so it is an important account. Admin rights are not enough for certain actions.

3) Once organization is created target country should invite FMI developer(s) to the organization as admins.

4) FMI developers with admin rights can then continue with creating the app and adding required certificates etc.

5) A project needs to be created to Google Cloud, but information how to do that is provided when it is needed (guide is very specific so it's not appropriate to add it to this initial guide).
